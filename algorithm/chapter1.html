Design and Analysis of Algorithms

The Role of Algorithms in Computing

알고리즘 정의 :

  • 한정된 시간 내에 어떤 값을 또는 값 집합을 입력으로 받아 어떤 값을 또는 값 집합을 출력으로 생성하는 잘 정의된 계산 절차
  • 입력을 출력으로 변환하는 계산적인 단계의 연속
  • 잘 정의된 계산 문제를 해결하기 위한 도구
    • ex) sorting problem

계산 문제에 대한 알고리즘은 올바르다면...

  • 입력으로 제공된 모든 문제 인스턴스에 대해 알고리즘이 중단(유한한 시간 내에 계산을 완료)
  • 올바른 해결책을 출력

올바르지 않은 알고리즘은 오류율을 제어할 수 있다면 여전히 유용 (올바른 답변과 유사 / 올바른 알고리즘보다 빠름)


Related Concepts: Data Structures

정의 : 데이터를 저장하고 정리하여 액세스와 수정을 용이하게 하는 방법 (적절한 데이터 구조 사용 -> 알고리즘 설계의 중요한 부분)


Related Concepts: Hard Problems

빠른 알고리즘을 가지지 않는것으로 알려진 문제(NP-complete problem)

ex) traveling salesman problem(모든 경우의 수 탐색)


Related Concepts: Alternative Computing Models

병렬 컴퓨팅

일부에서 알고리즘을 시작할 때 입력이 사용불가능 대신, 입력은 시간이 지남에 따라 도착하며, 알고리즘은 미래에 어떤 데이터가 도착할지 알지 못한 채로 어떻게 진행할지 결정 데이터 센터에서 작업이 계속해서 도착하고 떠나며, 스케줄링 알고리즘은 미래에 어떤 작업이 도착할지 알지 못한 채로 어떤 작업이 언제 어디서 실행될지를 결정해야 함


Algorithms as a technology

Efficiency(효율성)

동일한 문제를 해결하기 위해 고안된 다양한 알고리즘은 종종 효율성 면에서 크게 다를 수 있다. (ex Insertion sort vs Merge sort)
