3.1 O-notation, Ω-notation, and θ-notation

Asymptotic Notations

입력 크기에 따른 알고리즘의 실행시간

  • Asymptotic notation: O-notation

    upper bound

    O(g(n)) = {f(n) : there exist positive constants c and n0 such that
       0 <= f(n) <= cg(n) for all n >= n0}

  • Asymptotic notation: Ω-notation

    lower bound

    Ω(g(n)) = {f(n) : there exist positive constants c and n0 such that
       0 <= cg(n) <= f(n) for all n >= n0}

  • Asymptotic notation: θ-notation

    tight bound

    θ(g(n)) = {f(n) : there exist positive constants c1, c2 and n0 such that
       0 <= c1g(n) <= f(n) <= c2g(n) for all n >= n0}



  • o-notation

    upper bound that is not asymptotically tight.

    o(g(n)) = {f(n) : for any positive constants, there exists a constant n0 > 0 such that
       0 <= f(n) < cg(n) for all n >= n0}

    f(n) is asymptotically smaller than g(n)

  • ω-notation

    a lower bound that is not asymptotically tight

    ω(g(n)) = {f(n) : for any positive constants, there exists a constant n0 > 0 such that
       0 <= cg(n) < f(n) for all n >= n0}

    f(n) is asymptotically larger than g(n)

Standard Notations and Common Functions

Monotonicity

  • A function f(n) is monotonically increasing if m <= n implies f(m) <= f(n)
  • A function f(n) is monotonically decreasing if m <= n implies f(m) >= f(n)
  • A function f(n) is strictly increasing if m < n implies f(m) < f(n)
  • A function f(n) is strictly decreasing if m < n implies f(m) > f(n)
